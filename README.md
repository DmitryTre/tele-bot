# KittyBot

Telegram‑бот, который отправляет милые картинки котов и собак, а также комплименты. Идеален для поднятия настроения!


## Функционал

Бот реагирует на следующие команды:

* `/start` — приветствие, показ клавиатуры с командами и отправка фото кота и собаки;
* `/newcat` — отправка случайного фото кота;
* `/newdog` — отправка случайного фото собаки;
* `/compliment` — отправка случайного комплимента.


## Технологии

* Python 3.x
* `pyTelegramBotAPI` (библиотека `telebot`) — для взаимодействия с Telegram Bot API;
* `requests` — для запросов к внешним API;
* `python-dotenv` — для загрузки переменных окружения из `.env` файла.

## Внешние API

* **The Cat API** (`api.thecatapi.com`) — источник фото котов;
* **The Dog API** (`api.thedogapi.com`) — источник фото собак;
* **Compliment API** (`tools-api.robolatoriya.com/compliment`) — источник комплиментов.

## Установка и запуск

### Шаг 1. Клонирование репозитория

bash
git clone https://github.com/yourusername/kittybot.git
cd kittybot

### Шаг 2. Создание виртуального окружения (опционально, но рекомендуется)
bash
python -m venv venv
source venv/bin/activate  # Linux/MacOS
ИЛИ
venv\Scripts\activate  # Windows

### Шаг 3. Установка зависимостей
bash
pip install -r requirements.txt

### Шаг 4. Настройка переменных окружения
Создайте файл .env в корневой директории проекта.

Добавьте в него ваш токен Telegram‑бота:

env
TOKEN=ваш_токен_от_BotFather
Чтобы получить токен:

напишите @BotFather в Telegram;

выполните команду /newbot;

следуйте инструкциям и скопируйте полученный токен.

### Шаг 5. Запуск бота
Запустите основной скрипт:

bash
python kittybot.py
Бот начнёт polling и будет готов к работе!

Как это работает
Пользователь отправляет команду боту (например, /newcat).

Бот вызывает соответствующую функцию (get_new_cat()).

Функция делает запрос к внешнему API и получает URL изображения.

В случае ошибки при запросе к основному API (The Cat API) функция автоматически переключается на резервный источник (The Dog API).

Бот отправляет полученное фото или текст в чат с пользователем.

Особенности реализации
Отказоустойчивость: при ошибке загрузки кота бот показывает собаку (и наоборот).

Логирование: все ошибки записываются в лог.

Удобная навигация: после команды /start пользователь видит клавиатуру с кнопками для быстрого доступа к командам.
